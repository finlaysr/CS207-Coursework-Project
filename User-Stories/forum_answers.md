# Requirements  
  
* Player names and stats are stored (not clear if need to log in with password or not)  
* Only the state of the current game is saved  
* During the game the player can ask for a hint or see the solution   
* All cryptograms have equal difficulty   
* The game should only show feedback after all letters have been entered  
* All guesses, including incorrect guesses should be put on the board  
* A letter can only be used as a guess once  
* Each player can have a save file for their saved cryptograms  
* When the game starts either:  
    * The user will get a random cryptogram, or  
    * They will be able to choose a pre made one from a list  
    * (We get to choose which one to implement)  
* The player should get to choose if they want “letter to letter” or a “letter to number” cryptogram for each new game. Not able to choose type of cypher though.   
* The scoreboard should only show the top 10 players ranked by the number of cryptograms they have completed successfully   
* In addition to the puzzle, the UI should also show a list of letter frequencies in the puzzle, timer not needed  
* Features such as font size and high contrast mode not needed  
* The players should be able to view their personal stats, such as number of cryptograms they have completed successfully   
* There is no limit to the amount of mistakes the players can make in a game  
* Players are not penalised for asking for a hint  
* Player stats need to be stored in persistent storage across runs of the program
